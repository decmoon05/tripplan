# 보안 가이드

> 이 매뉴얼은 인증/인가, 입력 검증, 보안 취약점 방지 시 참조한다.
> 관련 매뉴얼: `backend.md`, `error-handling.md`

## 인증 (Authentication)

- **Supabase Auth를 사용한다** (JWT/bcrypt 직접 구현 금지)
- `@supabase/ssr` 패키지로 쿠키 기반 세션 관리
- `src/middleware.ts`에서 모든 요청에 대해 세션 갱신 처리
- 비밀번호 해싱은 Supabase 내부에서 처리 (직접 구현 금지)
- 소셜 로그인(Google, Kakao 등)도 Supabase Auth 사용
- Route Handler에서 인증 확인: `supabase.auth.getUser()` 사용 (`getSession()`은 토큰 검증 안 함)
- 평문 비밀번호를 로그, 응답, DB에 저장 절대 금지

## 인가 (Authorization)

- 모든 API 엔드포인트에 인가 확인 필수
- Route Handler에서 `supabase.auth.getUser()`로 사용자 확인 후 자원 소유자 검증
- Supabase RLS(Row Level Security) 정책을 DB 레벨에서 추가 적용
- 역할 기반 접근 제어가 필요한 경우 Supabase의 커스텀 클레임 활용

## 입력 검증

- 모든 사용자 입력은 서버에서 재검증 (클라이언트 검증만 의존 금지)
- 검증 라이브러리 사용 (수동 정규식 최소화)
- 파일 업로드: 확장자, MIME 타입, 파일 크기 모두 검증

## 개인정보 보호

### 개인정보보호법 대응

> **Tripplan이 수집하는 MBTI, 식성, GPS 데이터, 여행 일정은 모두 개인정보에 해당.**
> **위반 시 과징금 매출액 3%, 최대 5천만 원 과태료, 고의 유출 시 5년 이하 징역.**

#### 필수 구현 사항
1. **개인정보처리방침 페이지** — 수집 항목, 목적, 보유 기간, 제3자 제공, 파기 방법 명시
2. **동의 절차 UI** — 필수 동의(서비스 이용)와 선택 동의(마케팅) 구분
3. **위치정보 동의** — GPS 사용 전 별도 동의 팝업
4. **식성/건강 정보** — 민감정보에 해당할 수 있으므로 별도 동의 검토
5. **데이터 보유 기간** — GPS: 세션 종료 시 파기, 여행 일정: 1년, 결제 기록: 5년
6. **열람/수정/삭제 요청** — 사용자의 개인정보 처리 요청 기능 구현 필수

#### 코드 작성 시 체크리스트
- [ ] 개인정보 수집 전 동의 여부 확인 로직
- [ ] 보유 기간 만료 데이터 자동 파기 로직 (cron 또는 Supabase Edge Function)
- [ ] 개인정보 로깅 금지 (email, 위치, 식성 등이 로그에 노출되지 않도록)
- [ ] API 응답에 불필요한 개인정보 포함 금지 (필요한 필드만 반환)

### 위치기반서비스사업자 신고
- GPS/위치 데이터를 수집·이용하는 서비스는 **서비스 개시 후 1개월 이내** 방송통신위원회에 신고 필수
- **미신고 제재: 3년 이하 징역 또는 3천만 원 이하 벌금**

## 취약점 방지 체크리스트

- [ ] SQL Injection: 파라미터 바인딩 사용
- [ ] XSS: 사용자 입력 이스케이프 처리
- [ ] CSRF: 토큰 검증 또는 SameSite 쿠키
- [ ] 경로 조작: 파일 경로에 사용자 입력 사용 금지
- [ ] 민감 정보 노출: 에러 메시지에 스택 트레이스 포함 금지
- [ ] Rate Limiting: 로그인, 회원가입 등 주요 엔드포인트에 적용

## 환경 변수 관리

- 비밀키, DB 접속 정보 등은 반드시 환경 변수로 관리
- `.env` 파일은 `.gitignore`에 등록
- `.env.example`에 키 이름만 기록 (값은 비움)

- 하드코딩된 비밀키/비밀번호 금지
