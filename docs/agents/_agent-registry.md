# 전문 에이전트 레지스트리

> **이 파일의 목적**: 상황에 맞는 전문 에이전트를 빠르게 찾기 위한 라우팅 테이블.
> 각 에이전트의 상세 프롬프트는 개별 파일을 참조.
> 필요한 에이전트만 해당 파일을 열 것. 전체를 읽지 말 것.

---

## 에이전트 호출 조건표

| 상황 | 호출할 에이전트 | 파일 | 비고 |
|------|----------------|------|------|
| 기능 구현 완료 후 코드 리뷰 필요 | Review Agent | `review-agent.md` | |
| 테스트 코드 작성 필요 | Test Agent | `test-agent.md` | |
| 인증/인가/개인정보 관련 코드 작성 후 | Security Agent | `security-agent.md` | |
| 전체 품질 검사, 에러 다수 발견 시 | QA Agent | `qa-agent.md` | |
| 큰 기능 설계, 아키텍처 결정 시 | Planning Agent | `planning-agent.md` | 참조: `docs/prd/tripplan-prd.md` |

## 에이전트 사용 규칙

1. **단독 호출**: 한 번에 1개 에이전트만 호출. 순차적으로 진행.
2. **보고서 필수**: 모든 에이전트는 반드시 보고서 형식으로 결과를 출력해야 한다.
3. **상호 리뷰**: 구현 에이전트와 리뷰 에이전트는 반드시 다른 세션/에이전트로 분리.
4. **매뉴얼 참조**: 각 에이전트는 자신의 프롬프트에 명시된 매뉴얼만 참조.

## 에이전트 요약

| 에이전트 | 핵심 역할 | 보고서 형식 |
|----------|-----------|-------------|
| **Review** | 코드 품질, 컨벤션, 구조 검토 | 발견/수정/근거 |
| **Test** | 단위/통합 테스트 작성 | 커버리지/테스트케이스 목록 |
| **Security** | 보안 취약점 탐지 | 위협/위험도/대응방안 |
| **QA** | 종합 품질 검사, 대량 에러 처리 | 검사항목/결과/수정이력 |
| **Planning** | 기능 설계, 기술 선택 | 설계안/대안비교/결정근거 |

## 호출 방법

Claude Code의 Agent 도구를 사용하여 해당 에이전트 프롬프트 파일의 내용을 프롬프트로 전달한다.

```
예시: Review Agent 호출
→ docs/agents/review-agent.md 파일을 읽고
→ Agent 도구의 prompt에 해당 내용 + 검토 대상 파일 목록을 전달
```
