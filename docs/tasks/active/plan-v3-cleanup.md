> 작성일: 2026-04-02
> 상태: 계획 수립 중

## 현재 문제 (솔직하게)

20사이클 반복했지만 100% PASS 한 번도 못 나옴. 80%가 최고.
같은 구조에서 파라미터 조정(오후 3개 제한, 저녁 범위 확장 등) 반복 → 근본 해결이 아닌 땜질.

### 구조적 문제

1. **v2 후처리가 v3 위에 덮어씀**
   - v3: slotAssigner가 배치 → timeCalculator가 시간 계산 (결정적)
   - 그런데 pipelineV3.ts line 118~119:
     ```
     tripItems = validateClosedDays(tripItems, input.startDate);  ← v2 함수
     tripItems = validateGeoBoundary(tripItems);                  ← v2 함수
     ```
   - v2 후처리가 v3가 배치한 식당을 삭제 → 저녁 누락

2. **실패 시 재요청 없음**
   - 논문: AI 추천 → 알고리즘 배치 → 검증 → **실패 시 AI에게 "이 장소 대신 다른 거 줘" 재요청**
   - 우리: AI 추천 → 배치 → 검증 → 실패해도 **그냥 반환**
   - 제주 327km 좌표 → 데이터 문제인데 AI에게 "제주 내 장소로 다시 줘"를 안 함

3. **데이터 인프라 불완전**
   - Nominatim: 제주 장소 검색률 ~60% (한국 소도시/특수 장소 약함)
   - Overpass: 영업시간 30% 커버리지
   - OSRM: 프로덕션 금지, driving만
   - → 데이터가 없으면 코드가 아무리 정교해도 무의미

## 해야 할 것

### 1. v2 후처리 완전 제거
- `validateClosedDays` → slotAssigner Phase 1에서 이미 처리됨. 중복 제거
- `validateGeoBoundary` → placeEnricher에서 좌표 확인 후 필터링. 중복 제거
- `augmentMissingMeals` → v3에서 사용 안 함 확인

### 2. 실패 시 AI 재요청 루프 추가
```
Phase 6.5 (신규):
  검증 실행 → 실패 항목 수집
  if 실패 > 0:
    AI에게: "이 장소들이 문제야: [목록]. 대체 장소 줘"
    대체 장소로 교체 → 재검증
    최대 1회 재시도
```

### 3. 사용자 의도 반영 구조
- 시간 고정(18:00 저녁 강제) 제거
- 사용자 specialNote에서 의도 파싱 (이미 extractMealPreferences 있음)
- 검증 기준도 사용자 의도에 따라 동적 변경
