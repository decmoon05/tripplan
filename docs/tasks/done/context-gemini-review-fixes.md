## 1. 왜 이렇게 결정했는가

| 결정 사항 | 선택지 | 최종 선택 | 이유 |
|-----------|--------|-----------|------|
| 장소 데이터 캐싱 | DB 전체 저장 vs place_id만 | place_id만 저장 | Google ToS 명시적 금지, API 키 정지 위험 |
| 결제 연동 | 불가능 판정 vs PG사 연동 | PG사 연동 가능 | 여신전문금융업법은 PG사 운영에 적용, 이용은 사업자등록증만 필요 |
| 사업자 등록 유형 | 정보통신업 확정 vs 법률 검토 | 법률 상담 후 결정 | 관광진흥법 "알선" 해석 여지 존재 |
| 식사 비용 추정 | price_level 사용 vs AI 리뷰 분석 | AI 리뷰 텍스트 분석 | price_level은 상대적 enum, 금액 아님 |

## 2. 제약 조건

- 기술적: Google Places API ToS (place_id 외 캐싱 금지)
- 법적: 개인정보보호법, 관광진흥법, 위치기반서비스사업법
- 비용: 사용자당 API 호출 비용 $0.30~$1.00 → 2,900원 수익 모델과의 균형

## 3. 참고 자료 위치

| 자료 | 위치 |
|------|------|
| Google Places API ToS | https://developers.google.com/maps/terms |

| 위치기반서비스사업법 | 법률 외부 참조 |

## 4. 주의 사항 및 함정

- Google Places API: 세션 캐시(30분)는 허용되나 영구 저장은 위반
- price_level: INEXPENSIVE~VERY_EXPENSIVE 4단계뿐, 국가/지역별 기준 상이
- iOS PWA: 푸시 알림 iOS 16.4+만, 백그라운드 GPS 불가
- 위치기반서비스사업자: 서비스 개시 후 1개월 이내 신고, 미이행 시 형사처벌
