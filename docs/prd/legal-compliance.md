# Tripplan 법적 준수 사항

> 작성일: 2026-03-08
> 버전: 1.0
> 상태: 검토 필요 (법률 전문가 상담 전)

## 1. 사업자 등록 유형

### 현황
- Gemini는 "절대 여행업으로 등록하면 안 된다, 정보통신업으로 등록하라"고 조언
- **이 조언은 단정적이며 위험함**

### 쟁점: 정보통신업 vs 여행업

| 구분 | 정보통신업 | 여행업 (관광진흥법) |
|------|-----------|-------------------|
| 등록 난이도 | 낮음 (사업자등록만) | 높음 (보증보험, 자본금 요건) |
| 적용 법률 | 정보통신법 | 관광진흥법 |
| Tripplan 해당 여부 | AI 여행 정보 제공 → 해당 가능 | 숙소/항공 알선 수수료 → 해당 가능 |

### 회색지대 분석
- **관광진흥법 제3조:** "여행업"이란 여행자를 위해 시설 이용 알선, 운송·숙박 등의 서비스를 제공하는 업
- Tripplan이 단순 정보 제공만 하면 → 정보통신업 가능
- Tripplan이 숙소/항공 예약 제휴 수수료를 받으면 → "알선"으로 해석될 여지 있음

### 권장 조치
1. **MVP 단계 (Phase 1~2):** 정보통신업으로 등록 (제휴/수수료 없이 정보 제공만)
2. **Phase 3 (수익화) 전:** 관광법 전문 변호사 상담 필수
3. **제휴 수수료 모델 도입 시:** 여행업 등록 여부 법률 검토

> **절대 "정보통신업이면 안전하다"고 단정하지 말 것. 사업 모델이 변할 때마다 법적 검토 필요.**

## 2. 위치기반서비스사업자 신고

### 법적 근거
- **위치정보의 보호 및 이용 등에 관한 법률 (위치정보법)**
- GPS, IP, WiFi 등으로 위치정보를 수집·이용하는 서비스 제공 시 신고 의무

### Tripplan 해당 여부
- 사용자 위치 기반 주변 장소 추천 → **해당**
- 여행 일정에 위치 데이터 활용 → **해당**

### 신고 요건

| 항목 | 내용 |
|------|------|
| 신고 시기 | 서비스 개시 후 **1개월 이내** |
| 신고 기관 | 방송통신위원회 |
| 필요 서류 | 사업자등록증, 위치기반서비스 내용 설명, 개인정보처리방침 등 |
| **미이행 제재** | **3년 이하 징역 또는 3천만 원 이하 벌금** |

### 조치 계획
- [ ] MVP 개발 완료 시점에 신고 준비
- [ ] 서비스 런칭 후 1개월 이내 신고 완료
- [ ] 위치정보 이용약관 작성

## 3. 개인정보보호법

### Tripplan이 수집하는 개인정보

| 데이터 | 개인정보 여부 | 근거 |
|--------|-------------|------|
| 이메일 | O | 식별정보 |
| MBTI 스타일 | O | 개인 성향 정보 |
| 여행 페이스 | O | 개인 선호 정보 |
| 식성/식이제한 | O | 건강 관련 정보 (민감정보 가능성) |
| GPS 위치 | O | 위치정보 |
| 여행 일정 | O | 행동 패턴 정보 |
| 결제 정보 | O | 금융 정보 (PG사 위탁) |

### 법적 필수 조치

#### 3.1 개인정보처리방침 작성 및 공개
- 수집 항목, 수집 목적, 보유 기간 명시
- 제3자 제공 내역 (Google, AI API 제공자)
- 위탁 처리 내역 (PG사, 클라우드 서비스)
- 파기 절차 및 방법
- **웹사이트/앱 내 상시 접근 가능하도록 게시**

#### 3.2 동의 절차
- 회원가입 시 개인정보 수집·이용 동의 (필수)
- 위치정보 수집·이용 동의 (필수 - 위치 기능 사용 시)
- 마케팅 정보 수신 동의 (선택)
- **식성/건강 정보는 민감정보에 해당할 수 있으므로 별도 동의 검토**

#### 3.3 보유 기간 정책

| 데이터 | 보유 기간 | 근거 |
|--------|-----------|------|
| 계정 정보 | 탈퇴 후 30일 | 재가입 방지 |
| 여행 일정 | 생성 후 1년 (또는 사용자 삭제 시) | 서비스 이용 목적 |
| 결제 기록 | 5년 | 전자상거래법 |
| GPS 데이터 | 세션 종료 시 파기 | 최소 수집 원칙 |

#### 3.4 위반 시 제재
- **과징금:** 매출액의 최대 3%
- **과태료:** 최대 5천만 원
- **형사처벌:** 고의 유출 시 5년 이하 징역

### 데이터 거주지 (국외 이전)
- Supabase는 한국 리전을 제공하지 않음 → **도쿄(ap-northeast-1)** 또는 **싱가포르(ap-southeast-1)** 선택
- 개인정보의 국외 이전에 해당하므로 **개인정보처리방침에 국외 이전 사실, 이전 국가, 수탁자(Supabase Inc.) 고지 필수**
- 동의 시 국외 이전 동의도 포함되도록 약관 설계

### 구현 체크리스트
- [ ] 개인정보처리방침 페이지 생성
- [ ] 회원가입 시 동의 UI 구현 (필수/선택 구분)
- [ ] 위치정보 동의 팝업 구현
- [ ] 데이터 보유 기간 자동 파기 로직
- [ ] 개인정보 열람/수정/삭제 요청 처리 기능
- [ ] 개인정보 처리 위탁 계약 (Supabase, Google, AI 제공자)

## 4. Google Places API 이용약관

### 금지 사항
- `place_id`를 제외한 모든 장소 데이터의 영구 저장/캐싱 **금지**
  - 장소명, 평점, 리뷰, 영업시간, 사진 URL 등
- Google 로고/저작자 표시 누락 금지
- API 데이터를 Google Maps 외 지도에서 표시 금지

### 허용 사항
- `place_id` 영구 저장: **허용**
- 클라이언트 측 세션 캐시 (30분): **허용**
- 사용자가 직접 입력한 메모/코멘트: **저장 가능** (API 데이터 아님)

### 구현 규칙
- DB에는 `place_id`와 생성 시점 스냅샷(표시용 이름)만 저장
- 상세 정보 조회는 항상 실시간 API 호출
- 클라이언트에서 세션 스토리지로 30분 캐시

## 5. 표시광고법 (Phase 3)

- 광고 콘텐츠는 "광고" 또는 "AD" 명확 표시
- AI 추천 장소 중 제휴 업체는 "제휴" 표시 필수
- 리뷰/평점 조작 금지

## 6. 결제 관련 법률 (Phase 3)

### 여신전문금융업법 → Tripplan에 직접 적용 안됨
- 이 법은 PG사(결제대행사)를 **운영**하려는 기업에 적용
- PG 서비스를 **이용**하는 것은 사업자등록증만 있으면 가능

### 전자상거래법
- 통신판매업 신고 필요 (건별 결제 서비스 제공 시)
- 청약 철회(환불) 규정 준수
- 결제 기록 5년 보관

### PG사 연동 옵션

| PG사 | 특징 | 소규모 사업자 지원 |
|------|------|-------------------|
| 토스페이먼츠 | 간편 연동, 개발자 친화적 | O (개인사업자 가능) |
| NicePay (포스타트) | 소규모 특화 | O |
| KG이니시스 | 대형, 안정적 | △ (심사 다소 엄격) |
