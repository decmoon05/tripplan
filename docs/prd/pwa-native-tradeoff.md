# PWA vs 네이티브 트레이드오프 분석

> 작성일: 2026-03-08
> 버전: 1.0

## 1. 배경

Gemini는 "Next.js + PWA가 초기 MVP 배포 속도 면에서 가장 옳다"고 했으나, 여행 앱의 핵심 기능에 대한 PWA 제한사항을 언급하지 않았음.

## 2. PWA vs 네이티브 비교

| 기능 | PWA (Next.js) | 네이티브 (React Native / Flutter) |
|------|--------------|----------------------------------|
| **개발 속도** | 빠름 (웹 기술) | 느림 (플랫폼별 대응) |
| **배포** | URL 즉시 배포 | 앱스토어 심사 (1~7일) |
| **설치 없이 사용** | O (브라우저 접속) | X (앱 설치 필요) |
| **푸시 알림 (Android)** | O | O |
| **푸시 알림 (iOS)** | △ (iOS 16.4+ 만) | O |
| **백그라운드 GPS** | **X** | O |
| **오프라인 지도** | △ (제한적 캐시) | O (전체 지도 다운로드) |
| **오프라인 일정** | △ (서비스 워커) | O (로컬 DB) |
| **카메라/갤러리** | △ (제한적) | O |
| **네이티브 UX** | △ (웹 느낌) | O |
| **홈 화면 추가** | △ (직관적이지 않음) | O (앱스토어에서 설치) |
| **앱스토어 노출** | X | O (검색 유입) |
| **비용** | 낮음 | 높음 (2~3배) |

## 3. 여행 앱에서 특히 중요한 기능

### 3.1 오프라인 사용 (해외 여행 시 필수)
- **문제:** 해외에서 데이터 로밍 없이 일정 확인 필요
- PWA: 서비스 워커로 기본 캐시 가능하나 지도/장소 사진은 제한적
- 네이티브: 전체 일정 + 지도를 로컬에 저장 가능

### 3.2 위치 기반 기능
- **문제:** "주변 맛집 추천", "다음 장소까지 경로 안내" 등 실시간 위치 필요
- PWA: 포그라운드에서만 GPS 사용 가능, 백그라운드 추적 불가
- 네이티브: 백그라운드 GPS, 지오펜싱 가능

### 3.3 푸시 알림 (일정 리마인더)
- **문제:** "30분 후 다음 장소로 이동하세요" 등 시간 기반 알림
- PWA + iOS: iOS 16.4+ (2023년 3월 출시) 에서만 지원, 홈 화면에 추가한 경우만
- 네이티브: 모든 버전에서 완벽 지원

## 4. 결정: MVP는 PWA, 추후 네이티브 전환 검토

### PWA로 시작하는 이유
1. **개발 리소스:** 1인 개발자, 양 플랫폼 네이티브 개발은 비현실적
2. **검증 우선:** PMF(Product-Market Fit) 확인 전 네이티브 투자는 리스크
3. **배포 속도:** 앱스토어 심사 없이 즉시 업데이트
4. **비용:** 개발비/유지보수비 2~3배 차이

### PWA 제한사항 완화 전략

| 제한 | 완화 방법 |
|------|-----------|
| 오프라인 일정 | 서비스 워커 + IndexedDB로 일정 데이터 로컬 저장 |
| 오프라인 지도 | Google Maps 정적 이미지를 캐시 (일부 제한) |
| iOS 푸시 | 이메일/SMS 알림으로 대체 (MVP) |
| 홈 화면 추가 | 설치 배너 UI 구현, 가이드 팝업 제공 |

### 네이티브 전환 검토 시점
- DAU 10,000+ 달성
- 사용자 피드백에서 오프라인/GPS 요구가 상위 3개 내
- 투자 유치 또는 매출 안정화 후
- 전환 시 React Native 또는 Expo 권장 (Next.js 코드 일부 재사용 가능)

## 5. 중간 대안: Capacitor (하이브리드)

PWA와 네이티브의 중간 지점으로, 웹 코드를 네이티브 앱으로 래핑하는 방법.

| 항목 | Capacitor |
|------|-----------|
| 기반 | 기존 Next.js 코드 재사용 |
| 네이티브 기능 | 플러그인으로 GPS, 푸시 등 접근 |
| 앱스토어 배포 | 가능 |
| 추가 개발 비용 | PWA 대비 30~50% 증가 |
| 단점 | 성능이 완전한 네이티브보다 열등 |

**권장:** MVP는 PWA → Phase 2 후반에 Capacitor로 앱스토어 배포 검토
