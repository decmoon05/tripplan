## 1. 프로젝트 개요

**스택**: React Native (Expo) + Node.js/TypeScript + Claude API
**플랫폼**: 웹 + iOS + Android (단계적 출시)

자세한 기획: `docs/00_PROJECT_PLAN.md`

## 3. 작업 승인 프로세스 (절대 준수)

> 이 프로세스를 무시하고 바로 코드를 작성하는 것은 **규칙 위반**입니다.

### 큰 작업 (새 기능, 구조 변경, 새 파일 3개 이상)

```
1. 계획서 작성
   → 무엇을 만들 것인지, 어떤 파일을 수정할 것인지 먼저 정리
   → 사용자에게 제시하고 명시적 승인 대기

2. 승인 확인 후에만 시작
   → "진행해" / "OK" / "그렇게 해" 등 명확한 허가 신호 필요
   → 승인 없이 작업 시작 금지

3. 순차 실행 (한 번에 1~2개)
   → 한 작업 완료 → 보고서 작성 → 다음 작업
   → 여러 작업 병렬 진행 금지

4. 각 작업 완료 후 보고서 작성 (아래 형식)
```

### 작은 작업 (버그 수정, 단순 수정, 1~2개 파일)
- 바로 진행 가능
- 완료 후 보고서는 필수

## 5. 전문 에이전트 운영

복잡한 작업은 역할별 전문 에이전트로 분리하여 처리합니다.

### Plan 에이전트 (기획/설계 담당)
**사용 시점**: 새 기능 추가 전, 구조 변경 전
**역할**: 구현 방법 설계, 파일 구조 계획, 의존성 파악
**호출 방법**: "Plan 에이전트로 [기능명] 구현 계획 세워줘"
**보고 형식**: 파일 목록 + 단계별 작업 순서 + 예상 이슈

### Quality 에이전트 (코드 리뷰 담당)
**사용 시점**: 기능 구현 완료 후, 수정 파일 5개 이상일 때
**역할**: 코드 품질 검토, 보안 취약점 확인, 일관성 점검
**호출 방법**: "Quality 에이전트로 [파일 목록] 리뷰해줘"
**보고 형식**: 발견된 문제 목록 + 심각도(critical/warning/info) + 수정 제안

### Test 에이전트 (테스트 담당)
**사용 시점**: 중요 기능(인증, AI 호출, DB 쿼리) 구현 후
**역할**: 테스트 케이스 작성, 엣지 케이스 발견, 실행 확인
**호출 방법**: "Test 에이전트로 [기능명] 테스트 작성해줘"
**보고 형식**: 테스트 목록 + 통과/실패 결과 + 미커버 케이스

## 6. 절대 규칙 (위반 금지)

### 6-1. 코드 품질
- TypeScript strict 모드 사용. `any` 타입 사용 금지.
- 모든 API 호출은 try-catch로 감싸고 에러를 명확히 처리.
- 환경변수(API 키, DB 비밀번호)는 절대 코드에 직접 작성 금지. `.env` 사용.
- `.env` 파일은 절대 커밋 금지. `.gitignore`에 포함 확인.
- 함수 하나는 한 가지 역할만. 100줄 넘으면 분리 고려.

### 6-2. 보안
- 사용자 입력은 항상 검증 (SQL Injection, XSS 방지).
- Claude API 키는 반드시 서버 사이드에서만 사용. 클라이언트 노출 절대 금지.
- 개인정보(여행 성향, 취향 데이터)는 최소 수집 원칙.

### 6-3. 과잉 개발 금지
- 요청하지 않은 기능 추가 금지.
- 현재 Phase에 없는 기능은 TODO에 기록만 하고 구현하지 말 것.
- Phase 1 완성 전 Phase 2 기능 구현 금지.

## 7. 기술 스택 & 아키텍처

```
TripWise/
├── apps/
│   └── mobile/          # React Native (Expo) - 모바일 앱
├── packages/
│   ├── api/             # Node.js/TypeScript 백엔드
│   ├── shared/          # 공유 타입, 유틸리티
│   └── ai/              # AI 통합 레이어 (Phase 1)
├── docs/                # 기획·맥락·할일 문서
└── CLAUDE.md            # 이 파일
```

**DB**: PostgreSQL (Supabase 클라우드) + Prisma ORM
**캐시**: Redis (Upstash 클라우드, Phase 1 이후)
**API 통신**: REST (Phase 1)

## 8. Phase 1 범위 (현재 집중할 것)

MVP에서 구현할 기능만:
1. 사용자 성격 프로파일링 (고정 질문 + AI 추가 질문)
2. 여행 목적지 선택
3. AI 기반 일정 초안 생성 (동선 최적화)
4. 여행지 정보 카드 (Google Places 기반)
5. 여행 전 실시간 이슈 체크 (공사, 공휴일, 이벤트)

**Phase 1에 없는 것**: 항공권/숙소 예약 연동, 재정 바구니, 커뮤니티 기능.
